Pros

  • Easy to read. Custom variables allow developers to add meaningful names to provide context for their variable.
  • Easy to update. Since custom variables are targeted at the root, it allows developers to change specifications such as colours, spacing, and font families in one place and have it apply globally.
  • Responsive design. Developers can use custom variables to set preferred themes such as having a light mode option and a dark mode option. Variables can also be overwritten in media queries. For example, a calc() function can be used to calculate the new font size in desktop view.
  • Fallback values. Similar to how fonts can have a fallback, developers can also set fallback values for their variables.
  • Can be manipulated with Javascript. Developers can target custom variables in their Javascript. This container is coded with a custom variable. Try changing the colour of this container below!

Cons

  • Overuse. Custom variables are great for global specifications such as naming primary and secondary colours, but overuse may complicate the code.
  • Improper naming. With the freedom to label custom variables with anything, improper naming conventions may lead to confusing codes.
  • Impact on performance. Overuse of custom variables can slow down performance during browser rendering.
  • Browser compatibility. Internet Explorer currently does not support custom variables.
