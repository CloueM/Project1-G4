What are CSS Custom properties (also called CSS Variables)?

They are values that can be reused throughout your styles and defined once. When declaring your custom property, they always start with two dashes and are assigned a value (e.g. --main-color). This will also need to be used with the var() function. To store your values, it is common practice to define the variables on :root for easy management of colours, fonts, and spacing across your whole website.

Why Do They Matter?

Modernized CSS

Instead of relying on preprocessors like Sass, Custom Properties are built into CSS itself - faster, simpler and more powerful.

More simplicity with the way we write css

With Custom properties, developers have the freedom to create their own reusable values - CSS couldn't do something like this before.

Solves a Universal Problem

When custom properties weren't around, stylesheets were filled with random and repeated values. With Custom Properties, it makes every developer's life easier.

More Scalable

Building big websites becomes easier to update and organized with the simplified and consistent rules that custom properties bring.

They help developers work together

This matters because, as developers build big websites or projects, they can share the same properties/variables for colours, fonts and spacing. This promotes consistency in everyone's design while still allowing room for individual creativity.

Less errors and mistakes

Instead of retyping the values, reusing them helps prevent errors and broken designs.

Why We Use CSS Custom Properties

CSS custom properties, also known as CSS variables, allow developers to store reusable values — like colors, font sizes, spacing, or layout dimensions — in one place. This helps make styling consistent across a website and makes updates easier, because changing the variable automatically updates all elements that use it.

Practical Uses of CSS Custom Properties

  • Dark Mode: Easily switch themes by changing the variable values, instead of editing each element individually.
  • Colors: Store main brand colors in variables for consistent design throughout the website.
  • Fonts & Spacing: Quickly adjust typography and layout across the site by updating a few variables.
  • Responsive Design: Use variables for breakpoints and flexible sizing, making it easier to adapt for mobile or desktop screens.

Using the CSS custom properties improves maintainability, reduces errors, and makes websites more scalable and adaptable for future updates.

Pros

  • Easy to read. Custom variables allow developers to add meaningful names to provide context for their variable.
  • Easy to update. Since custom variables are targeted at the root, it allows developers to change specifications such as colours, spacing, and font families in one place and have it apply globally.
  • Responsive design. Developers can use custom variables to set preferred themes such as having a light mode option and a dark mode option. Variables can also be overwritten in media queries. For example, a calc() function can be used to calculate the new font size in desktop view.
  • Fallback values. Similar to how fonts can have a fallback, developers can also set fallback values for their variables.
  • Can be manipulated with Javascript. Developers can target custom variables in their Javascript. This container is coded with a custom variable. Try changing the colour of this container below!

Cons

  • Overuse. Custom variables are great for global specifications such as naming primary and secondary colours, but overuse may complicate the code.
  • Improper naming. With the freedom to label custom variables with anything, improper naming conventions may lead to confusing codes.
  • Impact on performance. Overuse of custom variables can slow down performance during browser rendering.
  • Browser compatibility. Internet Explorer currently does not support custom variables.
